Migrate this project from Hostinger MySQL + custom backend APIs to Supabase.

Goal:

* eliminate the current failing backend/database architecture
* remove MySQL dependency
* remove json-files fallback mode
* stop relying on broken /api routes for basic CMS data
* use Supabase as the primary database + auth system

---

## Stack requirements

Use:

* Supabase
* @supabase/supabase-js
* PostgreSQL (Supabase managed)
* Vite frontend

Remove:

* mysql/mysql2
* old Hostinger DB logic
* json-files mode
* fake fallback database
* broken MySQL connection code

---

## Tasks

### 1. Install Supabase client

Install:

* @supabase/supabase-js

---

### 2. Create centralized Supabase client

Create:

* src/lib/supabase.js

Using:

* VITE_SUPABASE_URL
* VITE_SUPABASE_ANON_KEY

---

### 3. Replace old API fetches

Replace ALL frontend calls like:

* fetch("/api/pages")
* fetch("/api/site-settings")
* fetch("/api/admin/login")

with direct Supabase queries.

---

### 4. Pages migration

Replace:
GET /api/pages

With:

```js
const { data, error } = await supabase
  .from("pages")
  .select("*")
  .eq("showInNav", true);
```

Use this throughout navigation/footer/page loading.

---

### 5. Site settings migration

Replace:
GET /api/site-settings

With direct Supabase query:

```js
const { data, error } = await supabase
  .from("site_settings")
  .select("*")
  .single();
```

---

### 6. Admin login migration

Replace custom backend login system with Supabase Auth.

Requirements:

* email/password login

* logout support
* protected admin routes

Use:

```js

```
