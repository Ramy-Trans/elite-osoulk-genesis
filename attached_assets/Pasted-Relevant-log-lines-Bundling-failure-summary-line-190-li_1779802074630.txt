Error type and cause

Error: "Top-level await is currently not supported with the 'cjs' output format" (esbuild bundling error).
Cause: The function bundle includes code (server/index.js) that uses top-level await (e.g. const { createServer } = await import('vite') and const vite = await createViteServer(...)) at module top-level. Netlify's bundling step emits a CommonJS bundle (cjs) by default; top-level await is only supported in ESM. This typically happens when dev-only Vite server code (createViteServer) or other ESM-only code is imported into a Netlify function that is being bundled for production.
Solution (concrete, actionable)

Inspect which function is importing the dev-server code.
Open netlify/functions/api.js and any files it imports. If it imports server/index.js (the file that contains the top-level await), remove that dev-only import.
If you need to view the server file referenced in the logs, here's the file in your repo: https://github.com/Ramy-Trans/elite-osoulk-genesis/blob/main/server/index.js#L1400-L1420
Preferred fix: stop including Vite dev-server or other code with top-level await in production functions.
Replace the dev-only import with production-ready code (use the built server entrypoint or the adapter for your framework).
If this is a SvelteKit/Vite project, use the production build output (dist/server or the adapter-netlify entry) instead of spinning up a Vite dev server inside a function.
If you must keep dynamic importing, move awaits out of module top-level into an async initialization function called inside the function handler. Example change (move top-level await into handler):
// bad: top-level await (causes esbuild cjs error)
const { createServer: createViteServer } = await import('vite');
const vite = await createViteServer({ server: { middlewareMode: true } });

// good: lazy init inside handler
let viteServerPromise;
async function getViteServer() {
  if (!viteServerPromise) {
    viteServerPromise = import('vite')
      .then(({ createServer }) => createServer({ server: { middlewareMode: true } }));
  }
  return viteServerPromise;
}

exports.handler = async (event, context) => {
  const vite = await getViteServer();
  // ...use vite in request handling...
};
The key is to avoid top-level await in module scope; performing dynamic import/initialization inside the request handler (or via a lazily-initialized promise) prevents esbuild from treating it as top-level await.
Alternative: if that code is not needed in the function at all, remove it or change your function to use the production server entry (for example, import the prebuilt server handler from your build output rather than importing vite).
Rebuild and redeploy.
